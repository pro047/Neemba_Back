## 4. 다음 세션 최우선: CI 테스트 게이트 (§7-2 #1)

**동기**: 현재 `deploy.yml`에는 **테스트 스텝이 전혀 없다** — 테스트가 깨져도 프로덕션까지 도달한다. (단, 이번 `--env-file` 버그는 유닛테스트로는 못 잡는 배포-설정 버그였음을 유념. CI 게이트는 "테스트 회귀"를 막는 것.)

### 착수 계획 (제안)
1. **별도 CI 워크플로** `.github/workflows/ci.yml` 신설 — 트리거: `pull_request` (base develop/main) + `push` (develop). 두 잡:
   - **node-test**: `actions/setup-node@v4`(node 20, cache npm, `services/node/package-lock.json`) → `cd services/node && npm ci && npm test && npx tsc --noEmit`
   - **python-test**: `actions/setup-python@v5`(3.12) + `astral-sh/setup-uv@v3` → `cd services/python && uv run --extra dev pytest tests/ -q` (필요 시 `py_compile`도)
   - (참고: `deploy.yml`에 이미 node/python/uv 셋업 스텝이 있으니 그 설정을 복사하면 됨.)
2. **배포 차단 연결**: 두 가지 중 택1 —
   - (A) `deploy.yml`의 deploy 스텝 **앞에** 동일 테스트 스텝을 인라인으로 추가(실패 시 배포 중단). 가장 단순·확실.
   - (B) CI 워크플로를 별도로 두고 branch protection rule로 develop/main 머지를 게이팅(설정은 GitHub UI/`gh api` 필요).
   - **권고**: 먼저 (A)로 deploy.yml에 테스트 게이트를 넣어 "깨진 코드가 배포되지 않음"을 즉시 보장하고, (B) branch protection은 후속.
3. TDD 대상이 아니므로(워크플로 설정) 로컬에서 `act` 없이도 논리 검증 후 PR. 머지 후 다음 배포에서 실효.

**주의**: deploy.yml은 `secrets.EC2_SG_ID`로 러너 IP를 SG에 임시 허용/회수한다. CI 워크플로는 서버 접속이 없으니 그 스텝 불필요(테스트만).

## 6. 미해결·주의사항

- **mvp 앱 수정**(토스트 순서 + rtmp mounted 가드): `~/Documents/translation/mvp` 워킹트리에 커밋 안 된 채 존재. 사용자가 feature/mic WIP 커밋 시 함께 올릴 예정. mvp에 main-develop 구조 세팅은 WIP 정리 후로 미룸.
- **RTMP ffmpeg 무경계 재시작(신규 백로그)**: OBS 등 업스트림이 없으면 ffmpeg가 `rtmp://neemba.app:1935`를 못 당겨 10초마다 무한 재시작(STT 경계·하트비트와 **같은 계열**). `FfmpegTranscoder.ts:28`의 pull URL이 **하드코딩**이라 dev/prod 분리도 안 됨 → env화 + 재시작 상한 + RTMP 스트림 키 인증(on_publish)을 묶어 처리 권장.
- **dev 스택 상태**: 실기기 테스트용으로 `.env.dev` WS_URL·앱 `.env.dev` HOST를 LAN IP로 바꿨다가 **localhost로 복구함**. 로컬 dev 컨테이너는 아직 LAN IP 설정으로 떠 있을 수 있으니 다음에 쓸 때 `docker compose -f docker-compose.dev.yml --env-file .env.dev up -d --force-recreate` 권장. `.env.dev`엔 postgres 서비스 추가됨(gitignore 로컬 파일).
- **NATS 비밀번호 제약**: 반드시 영문자로 시작(숫자 시작 시 nats.conf `$VAR` 치환이 숫자 리터럴로 파싱돼 기동 실패). 2026-07-12에 dev·prod 재회전됨. 로컬 `.env.*.bak-*` 백업 파일 정리 대상.
- **기존 백로그**(hand-off §4에서 이월): backpressure 부재(P1, 선택지 표 먼저), consumer ack 시점 이동(큐 적재 후 → 처리 완료 후), DeepL 클라이언트 재사용+비동기화, VAD 기반 rotation, Makefile/ESLint 설정 오류.
