## 1. 프로젝트 개요

실시간 한→영 설교/방송 번역 파이프라인. 데이터 흐름:

```
앱 마이크 PCM
 → Node /api/mic WS (services/node)
 → Google STT v2 스트리밍 (interim+final, 285초마다 스트림 rotation)
 → InterimChunkOrchestrator가 델타만 추출
 → NATS JetStream (transcript.session.<sessionId>)
 → Python consumer (services/python) — sequence 멱등 dedup
 → SentenceSeparator (KSS 문장분리, 버퍼링, 타임아웃 flush 2s)
 → DeepL 번역 (동기 호출 — 개선 백로그 있음)
 → FastAPI /ws → 앱  (+ 병렬: 마스킹 → Postgres 저장 + /ws/monitor 라이브)
```

세션 제어: 앱 → Node `POST /mic/start|stop` → Node가 Python `/internal/sessions/start|stop` HTTP 호출.

## 4. 앞으로 진행할 것 (우선순위 순)

### ① backpressure 부재 (P1 마지막) — **설계 선택지 먼저**
- 증상: Node `micWebSocket`·`googleSttV2`의 `write()` 반환값 무시(무한 버퍼), Python `websocket.py:75` 송신 태스크 무제한 생성, `sentence_queue`·`state_queue` 무제한.
- 구현 전에 선택지(예: drop-oldest vs 유한 큐+대기 vs 종단 역압)를 표로 정리해 사용자 결정 받을 것.

### ② P2 보안
- **NATS 자격증명 회전**: `infra/nats/nats.conf`에 `admin/admin`, `neemba/nats1234`가 git 히스토리 146커밋에 노출됨. 코드 수정만으론 부족 — **비밀번호 회전 필수**. 회전 시 `.env.prod`, Node `config.ts`(하드코딩 폴백 `nats1234` 제거), compose 동시 수정. 운영 중단 타이밍은 사용자와 협의.
- 저위험 선착수 가능: prod compose에서 NATS 모니터링 `8222:8222` 포트 제거, RTMP `1935` 인증/차단 검토, `reloader` 사이드카(docker.sock + 무인증 nc) 제거·대체.
- `/ws` 인증: 현재 sessionId는 라우팅 키일 뿐 인증 아님. 토큰 방식은 앱 수정 필요 → 사용자와 협의.
- `src/config.py:25` — DeepL 키·DB 비밀번호가 기동 로그에 평문 출력 (`mask=True` 누락). 소규모 수정.

### ③ 나머지 백로그

### 5-3. 런타임 검증 체크리스트 (이번 수정들이 실제로 동작하는지)

각 항목은 "수정한 PR ↔ 확인 방법" 매핑:

1. **첫 발화 (PR #4)**: 세션 시작 후 첫 문장이 앱/모니터에 표시되는가. 수정 전엔 100% 유실됐음.
2. **JetStream 설정 (PR #7)**: Python 기동 로그에서 `consumer: stream "..." exists/created`, `durable "..." exists/created` 확인. **기존 수동 생성 컨슈머가 있으면 `exists`로 뜨고 코드의 ack_wait=30s/max_deliver=5는 적용 안 됨** — 값 적용이 필요하면 기존 컨슈머 삭제 후 재기동.
3. **타임아웃 flush (PR #8)**: 종결어미 없이 말을 멈추고 2초 대기 → 미완성 문장이 번역되어 도착하는가. 값 조정은 `SentenceSeparator(flush_timeout_seconds=...)`.
4. **rotation 경계 (PR #6)**: 5분 이상 연속 발화 세션에서 285초 경계 전후 문장이 오염 없이 이어지는가 (경계 순간 단어 절단 자체는 남아 있음 — VAD 백로그).
5. **유령 세션 (PR #10)**: 마이크 연결 후 stop 없이 앱 강제 종료 → 10초 후 Node 로그에 `tearing down ghost session`, Google 스트림 종료·Python 세션 ended 확인. 10초 내 재접속하면 `teardown cancelled`.
6. **세션 stop 잔여 문장 (PR #8)**: 미완성 문장 상태에서 stop → 해당 문장이 DB(`app.translations`)와 모니터에 기록되는가.
7. **중복 차단 (PR #7)**: NATS를 잠깐 재시작하거나 DeepL을 지연시켜 재전달 유도 → 같은 문장이 두 번 나오지 않고 로그에 `duplicate dropped`.
8. **파이프라인 생존 (PR #5·#9)**: DeepL 키를 일부러 틀리게 해 기동 → 번역 실패 로그가 계속 찍히되 프로세스·태스크가 죽지 않는가. NATS 중단 → Node가 크래시하지 않고 `publish failed, span dropped` 로그.
9. **WS 게이팅 (PR #3)**: `/ws`에 sessionId 없이 접속하면 1008 거절. 세션 A stop이 세션 B의 소켓을 끊지 않는가.
10. **모니터링 회귀 (PR #1)**: `/monitor` Basic Auth, 히스토리 API, 30일 보존 컨테이너 정상.

측정 팁: end-to-end 지연 측정 코드는 아직 없음(감사 지적사항). 체감 확인은 발화 종료→앱 표시 시간을 스톱워치로.

### 5-4. 조정 예정 값 (사용자가 develop 테스트 후 결정)
| 값 | 위치 | 기본 | 의미 |
|---|---|---|---|
| `flush_timeout_seconds` | kss_separator.py 생성자 | 2.0s | 침묵 후 미종결 문장 flush |
